## Part 2: UI/UX Features

### Feature A: Interactive Bento Grid (blog index)
**Files:** `src/routes/blog.index.tsx`, `src/components/blog/article-card.tsx`

- First post (index 0) gets a 2-column span card with animated gradient border
- Use CSS `md:col-span-2` for the featured card
- Animated gradient border via `bg-[conic-gradient(from_var(--angle),aurora-1,aurora-2,aurora-3,aurora-1)]` with `@property --angle` CSS animation
- Rest remain single-column glass cards

### Feature B: Mouse-Tracking Radial Glow
**Files:** `src/components/blog/article-card.tsx` (new internal state)

- Add `onMouseMove` handler to each card
- Track cursor position as percentage from center
- Render a radial gradient overlay (`radial-gradient(circle at X% Y%, aurora-1/15, transparent 60%)`)
- Use `useRef` + `useState` for position, apply via inline style
- Subtle effect — only visible on hover

### Feature C: Proof-of-Work Micro-Badges
**Files:** `src/components/blog/article-card.tsx`

- Add optional `badges` field to `Article` type in `src/lib/blog.ts`
- Render small pill badges below the date: reading time, a hardcoded Lighthouse score, stack cost, and a metric
- Format: `<span className="text-[10px] font-mono ...">⏱ 5 min | 💰 $0/mo | 📈 40%↑</span>`
- For Medium articles, compute reading time from word count (already done); for local articles, parse from frontmatter or compute

### Feature D: Table of Contents Sidebar
**Files:** `src/routes/blog.$slug.tsx` (new `TableOfContents` component inline or extracted)

- Parse h2/h3 headings from markdown content using regex on the raw content string
- Generate slug IDs for each heading (lowercase, hyphenated)
- Render a sticky sidebar on desktop (`hidden lg:block fixed right-8 top-32`)
- On scroll, highlight the current section using `IntersectionObserver` on heading elements
- Mobile: collapsible TOC above the article (expand/collapse toggle)
- Style: glass card, monospace font, aurora accent on active heading

### Feature E: Contextual AI Chat
**Files:** `src/components/ai-chat.tsx`, `src/lib/chat.ts`

- Modify `sendChatMessage` server function to accept optional `articleContext` string
- When `articleContext` is provided, inject it into the Gemini system prompt: `You are answering questions about this specific article:\n\n${articleContext}\n\nAnswer based on the article content.`
- Modify `AiChat` component to accept optional `articleContext` prop
- In `blog.$slug.tsx`, render `<AiChat articleContext={article.content} />` instead of the global one
- Hide the global `<AiChat>` from `__root.tsx` when on a blog post page (check route), OR always show the global one but add the contextual one on blog pages
- **Simpler approach:** Pass article context to the existing global AiChat via React context or route context. The `sendChatMessage` function gains an optional `articleContext` param.

### Feature F: Better Code Blocks
**Files:** `src/routes/blog.$slug.tsx` (custom renderer)

- Create a custom `code` component for ReactMarkdown that:
  - Wraps `<pre><code>` in a container with a header bar
  - Header shows language label (extracted from `className="language-xxx"`) on the left
  - Copy button on the right (clipboard API, with check icon feedback)
  - Uses existing highlight.js atom-one-dark theme (no change to syntax highlighting)
- Style: rounded-xl, border, the existing `prose-pre` styles, plus a new header bar with `glass` styling
